## Система учета обращений
Этот проект — представляет собой трекер задач. Система позволяет пользователям создавать обращения, а администраторам — управлять ими, менять статусы, добавлять теги и оставлять ответы.

### Доступы для проверки
### Для быстрого тестирования функционала можно использовать следующие аккаунты:

- Администратор: Логин test, Пароль 00000000

- Клиент: Логин test2, Пароль 00000000

### Технический стек

- Backend: PHP 8+ (Native, без фреймворков), PDO для работы с БД.

- Frontend: jQuery (AJAX), Bootstrap 5, Select2.

- Архитектура: API-first подход, Composer (PSR-4), хранение настроек в .env.

### Как развернуть и запустить
### База данных

Используйте БД, дамп которой находится в корне проекта.

### Настройка проекта

Убедитесь, что установлен Composer.

Выполните команду в корне проекта:

<code>composer install</code>

Создайте файл .env и укажите данные вашей БД (пример оформления указан в .env.example):

```ini
DB_HOST=localhost
DB_NAME=support_tracker
DB_USER=root
DB_PASS=
```
### Запуск сервера. Проект настроен так, что входной точкой является папка public. Для запуска, можете как и я, использовать встроенный сервер PHP:

<code>php -S localhost:8000</code>
После этого откройте в браузере:

http://localhost:8000 (для клиента)

http://localhost:8000/admin (для администратора)

### Функциональные возможности для клиента:

- Регистрация и авторизация.

- Просмотр списка своих тикетов с фильтрацией и сортировкой.

- Форма создания нового обращения (модальное окно).

### Для администратора:

- Общая таблица всех обращений в системе.

- Управление статусами (ToDo, InProgress, Ready For Review, Done).

- Система тегов (tech issue, fatal error и др.) с использованием Select2.

- CRUD для тегов и статусов (управление справочниками напрямую из интерфейса).

- Защита от удаления: нельзя удалить статус, если он привязан хотя бы к одной активной задаче.
